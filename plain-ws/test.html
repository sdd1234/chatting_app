Plain WS — 시연 패널

로그인 → 자동 WS 접속/hello → 메시지 보내면 Redis가 어떻게 바뀌는지 한 화면에서 본다. 서버 archive 없음 — 채팅 archive는 이 브라우저 localStorage, Redis는 세션/오프라인 큐만.
user: (로그인 전) device: - plain-ws: CLOSED 공지: OFF 시연 메시지: 받는 사람이 오프라인이면 inbox에 적재 → 로그인 시 drain
📋 시연 흐름
  1. jihoon 로그인 → plain-ws 자동 접속 → session:jihoon:dev* + online에 jihoon 등장
  2. 다른 탭에서 또 jihoon 로그인 (같은 user, 다른 device) → user_devices:jihoon 2개 → 메시지 보내면 다른 디바이스에 carbon copy
  3. 로그인 안 한 emma에게 메시지 보내기 → inbox:emma List 적재 + TTL 7일
  4. 시크릿 탭에서 emma 로그인 → hello 직후 inbox drain → inbox:emma DEL
  5. 모든 jihoon 디바이스 닫기 → 마지막 디바이스 나가면 online에서 SREM + presence off
